JRecord 0.80.1

This version of JRecord adds getUniqueField & getUniqueFieldGroupsInSequence methods. These methods can be used to access fields via Group-Names and field names field names.

This is not a full release of JRecord, This version contains updated jars and source code. You should also download and install JRecord Version 0.80 if you have not already done so.

Background

In java and most languages, to access 

    a.b.c.d
you must specify all levels as above. Cobol is quite different, you only have to specify enough levels to uniquely qualify the field. Any of the following could be used: 
    d
    c.d
    a.c.d
    b.d
    a.b.c.d

This means

2         05 Customer-Details.
3            07 First-Name            pic x(20).
4            07 Last-Name             pic x(20).
5

Can be changed to

 8         05 Customer-Details.
 9            06 Customer-Name.
10               07 First-Name         pic x(20).
11               07 Last-Name          pic x(20).

Without affecting any Cobol programs. Most Cobol programmers would not even think about it. I have tried to implement group-level processing in a similar way in JRecord.

JRecord Details

To use the new features you will need a better understanding of JRecord than is required to just get fields by name.

JRecord has a schema definition (LayoutDetail): 

     LayoutDetail  - Describes a file
       |
       +----- RecordDetail (1 or More) - Describes one record in the file
                |
                +------  FieldDetail (1 or More)  - Describes one field in the file
It is on the RecordDetail class that 2 new methods (getUniqueField and getUniqueFieldGroupsInSequence) have been added. These new methods retrieve JRecords internal field-definition (FieldDetail). The FieldDetail can then be used to retrieve field values. 
   1 
   2         IFieldDetail companyNameFld = schema.getRecord(0).getUniqueField("COMPANY-NAME", "Company-Details");
   3         
   4         companyName = line.getFieldValue(companyNameFld).asString();
   5

Some key points
  1. You must retrieve the Record-Definition from the layout-definition (schema). For single record files, the record-index will be 0; for multi-record files you need to find the record-index.
  2. You must retrieve JRecords internal field-definition and use this to access field values.

Accessing Fields when there are duplicate field-names and duplicate group-names.

When there are both duplicate field and group names, you the group-sequence method getUniqueFieldGroupsInSequence

For the following copybook:

2            03 A.
3               05 B.
4                  07 Field-1    pic x(3).
5            03 B.                                       
6               05 A.
7                  07 Field-1    pic s9(6).
8            03 C.

You can access the fields by:

   1 
   2         IFieldDetail abField = record.getUniqueFieldGroupsInSequence("Field-1", "A", "B");
   3         IFieldDetail baField = record.getUniqueFieldGroupsInSequence("Field-1", "B", "A");
   4
